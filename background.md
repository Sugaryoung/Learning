我需要开发一个量化交易平台，支持A股回测与实盘模拟，同时保留未来扩展至加密货币的架构。需要有Web前端界面便于操作。

================================================================================
【一、技术栈要求】
================================================================================
后端：Python + FastAPI（提供REST API）
前端：Vue3 + Element Plus（或 React + Ant Design，你根据熟悉程度选择）
数据库：SQLite（存交易记录、回测结果），后续可换MySQL
数据处理：pandas, numpy
回测引擎：自定义事件驱动或基于backtrader（保留接口可换向量化回测）
实盘接入：A股使用 easytrader 或 xtquant（模拟打印即可），加密货币预留ccxt接口

================================================================================
【二、前端界面要求】
================================================================================
实现一个单页应用，包含以下页面/模块：

1. 策略管理页
   - 策略列表（展示已有策略名称、类型、创建时间）
   - 新增策略：选择模板（双均线、RSI、布林带等），设置参数（如快线周期、慢线周期）
   - 编辑/删除策略
   - 策略代码展示区（只读，后续可进阶为在线编辑）

2. 回测操作页
   - 选择策略（下拉框）
   - 选择股票代码（支持单个，如 000001.SZ）
   - 选择数据时间范围（日期选择器）
   - 设置初始资金、手续费率、滑点
   - 点击“开始回测”按钮，后端异步执行
   - 回测结果展示：
     - 关键指标卡片（总收益率、年化收益、最大回撤、夏普比率、胜率）
     - 资金曲线图（交互式，如ECharts）
     - 交易记录表格（买卖时间、价格、数量、盈亏）
     - 下载回测报告（CSV/PDF）

3. 实盘模拟页
   - 实盘账户信息（模拟：初始资金、当前净值、持仓列表）
   - 选择策略 + 股票 + 运行模式（定时轮询，如每30秒）
   - 显示实时信号（买入/卖出提示）
   - 订单记录表（已下单时间、方向、价格、状态）
   - 停止/启动按钮

4. 数据管理页（可选）
   - 手动导入CSV数据
   - 查看已存储的股票数据列表（代码、名称、最新日期）

5. 扩展预留页
   - 一个“市场切换”开关（A股 / 加密货币）
   - 加密货币模式下，交易对选择（BTC/USDT等），数据源切换至Binance API

界面风格：暗色背景+亮色图表，适合长时间盯盘。使用响应式布局。

================================================================================
【三、后端核心模块要求】
================================================================================
1. 数据模块 (data/)
   - 支持A股：从本地CSV或tushare/akshare获取（先用本地示例数据）
   - 统一OHLCV数据格式：timestamp, open, high, low, close, volume
   - 预留接口：CryptoDataHandler 继承 BaseDataHandler

2. 策略模块 (strategies/)
   - 策略基类：def init(self) / def next(self, bar, portfolio) → signal
   - 内置示例策略：双均线（SMA5/SMA20），RSI（超买超卖）
   - 策略参数可通过前端表单传入（JSON格式）

3. 回测引擎 (backtest/)
   - 事件驱动循环
   - 支持：初始资金、手续费(万2.5)、滑点(0.01元)
   - 记录每日持仓、交易明细
   - 返回结果：总收益率、夏普比率、最大回撤、交易列表

4. 实盘模拟模块 (live/)
   - 使用APScheduler定时执行
   - 调用策略 → 输出信号（只打印，不真实下单）
   - 模拟更新持仓和资金（简化：按信号价成交）
   - 支持停止/启动任务

5. API接口 (api/)
   - POST /backtest/run → 异步任务ID，轮询结果
   - GET /backtest/result/{task_id}
   - GET /strategies → 策略列表
   - POST /strategies → 保存新策略参数
   - POST /live/start → 启动实盘模拟任务
   - POST /live/stop → 停止任务
   - GET /live/status → 当前运行状态、持仓、资金
   - GET /markets/supported → 返回支持的交易市场列表（["A股","加密货币(预留)"]）

【四、具体示例策略（供AI参考）】
================================================================================
